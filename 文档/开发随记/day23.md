​	在个人成绩页面有一个按钮  “优秀学子”  ，即根据学分进行对所有学生排名（排行榜，只显示前x名）。

(比较平均分，定时任务将数据更新到缓存，利用Redis做排行榜（实时）)

​	关于这个排行榜，是否可以融合进入论坛功能呢，按负载程度排行课程

​	排行榜：

​		利用Redis的ZSet存储排行榜数据，以成绩作为分数（score），用户作为成员（member）

​		用户得分改变时，使用Redis的ZADD更新

​		使用ZREVRANGE命令按分数倒叙获取有序集合中的成员

​		ZSCORE获取某个用户的分数

​		ZREVRANK获取某个用户的排名

​	注意：

​		分数相同，排序规则？设计分数：最终分数 = 实际分数 + (时间戳偏移量)

​			假设用户得分是 `100`，当前时间戳是 `1719391200000`。你可以这样计算：

```
// 假设 maxTimestamp 是一个远大于当前时间戳的数，例如 2000000000000
double finalScore = userScore + (maxTimestamp - currentTimestamp) * 10e-10;
```

​			后达成相同分数的人，其“时间分”会更低。
